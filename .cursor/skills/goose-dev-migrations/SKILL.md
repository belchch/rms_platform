# goose — стратегия миграций (dev)

## Правило

До выхода в production **не создаём новые файлы миграций**. При любом изменении схемы:

1. Отредактировать `apps/api/migrations/001_init.sql` напрямую.
2. Сбросить и пересоздать dev-БД:

```bash
just db-reset
```

Внутри `just db-reset` выполняется:

```bash
goose -dir migrations postgres "$DATABASE_URL" down-to 0
goose -dir migrations postgres "$DATABASE_URL" up
```

## Когда начинать писать настоящие миграции

Когда проект выходит в beta/production и данные пользователей нельзя терять. До этого момента — только пересоздание.

## Что НЕ делать

- Не создавать файлы `002_...sql`, `003_...sql` на этапе разработки.
- Не писать `ALTER TABLE` в отдельных файлах — менять схему в `001_init.sql`.
- Не редактировать файлы в `internal/db/` вручную — они генерируются через `just api-gen`.
