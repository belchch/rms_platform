# Чеклист ревьюера

## 2026-05-10

- В сигнатуре handler-функции или поле struct handler'а нет `*db.Queries` — только `db.Querier`.
- Ветки handler'а, не вызывающие БД (парсинг, валидация, сравнение), вынесены в отдельные чистые функции либо имеют обоснование, почему остаются внутри.
- Если новый тест обоснован через "поднимет coverage до X%" — отклонить и попросить переформулировать через защищаемый инвариант.
- LWW-сравнения используют строгое неравенство (`>`), не `>=`.
- Каждое чтение `pgtype.Timestamptz` из БД проверяет `Valid` перед использованием `.Time`.
- Одна handler-функция не склеивает парсинг + валидацию + бизнес-решение + I/O без явного контракта на оркестрацию.
- Архитектурные изменения (новые границы слоёв, новые шовы) сопровождаются ADR в `docs/adr/`.
