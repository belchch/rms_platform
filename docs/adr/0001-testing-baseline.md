# ADR 0001: Контроль покрытия API через `just check`

## Статус

Принято — 2026-05-08 (коммит `5e52cc1`).

## Контекст

До этого момента в проекте не было механизма, который бы при обычном прогоне `just check` ловил регрессии по тестам бэкенда. Покрытие никем не измерялось, а часть тестов (`internal/handler/auth/handler_integration_test.go`) требовала живого Postgres из `.env` и поэтому падала вне рабочей машины автора.

Нужен был один порог, ниже которого `check` обязан падать, и одна команда для локального инспектирования покрытия — без отдельной CI-инфраструктуры.

## Решение

1. **Метрика — Go statement coverage**, посчитанный по всему `apps/api/...` минус сгенерированный sqlc (`internal/db/`).
   Фильтрация делается grep'ом по профилю (`coverage_filtered.out`), чтобы не раздувать числитель кодом, который мы не пишем руками.

2. **Два рецепта в `Justfile`:**
   - `just check-coverage` — `go tool cover -func` по отфильтрованному профилю. Локальный инструмент: посмотреть, какие пакеты непокрыты, прежде чем дописывать тесты.
   - `_api-coverage-threshold` (приватный) — встроен в `just check` и падает, если `total` ниже `api_coverage_min_pct` (сейчас `10`). Это и есть точка контроля регрессии.

3. **Интеграционные тесты изолированы build-тегом `//go:build integration`** и запускаются только через `just check-integration`. Иначе `just check` тащил бы за собой Postgres и переставал быть hermetic.

4. **Baseline зафиксирован** в `docs/planning/testing-baseline.md` (10.8%, разбивка по пакетам, endpoint coverage 1/6). Следующая цель — 30% statements и 50% endpoints.

## Последствия

- `just check` остаётся одним каноном проверки и не зависит от внешних сервисов.
- Порог `10%` — не финальный, а защита от движения вниз. Поднимаем его в том же коммите, в котором поднимаем покрытие. При этом **мы не стремимся к увеличению покрытия ради самого покрытия** — это всего лишь одна из метрик для анализа ситуации и защиты от регрессий.
- Любой новый тест, требующий БД, обязан нести тег `//go:build integration`, иначе CI начнёт требовать Postgres у всех.
- Переменная `api_coverage_min_pct` в `Justfile` — единственное место, где живёт текущий порог. Менять только её.
